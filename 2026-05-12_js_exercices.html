📖 Rappels essentiels — JavaScript moderne (ES6+)

const / let

const déclare une variable dont la liaison ne peut pas changer (≠ immuable). let permet de réassigner. Évite var : sa portée est capricieuse.

const pi = 3.14        // ne peut pas être réassigné
let compteur = 0
compteur = compteur + 1  // OK

Arrow functions

Syntaxe courte pour les fonctions. Si le corps est une seule expression, le return est implicite.

const double = (n) => n * 2
const saluer = (prenom) => `Bonjour, ${prenom} !`

Destructuring

Extraire des valeurs d'un tableau ou d'un objet directement dans des variables.

// Tableau
const [premier, deuxieme] = [10, 20, 30]

// Objet
const { nom, age } = { nom: "Alice", age: 30, ville: "Paris" }

Spread operator ...

Étale les éléments d'un tableau ou les propriétés d'un objet.

const base = [1, 2, 3]
const etendu = [...base, 4, 5]   // [1, 2, 3, 4, 5]

const user = { nom: "Bob" }
const userComplet = { ...user, role: "admin" }

Les modules ES6 (import/export) seront abordés dans les exercices suivants. Pour l'instant, travaille dans un seul fichier ou dans la console du navigateur.

EX 05

Mini-projet — Gestionnaire de contacts

moyen

Crée un mini-gestionnaire de contacts en utilisant tous les concepts vus aujourd'hui.

// Point de départ
const contacts = [
  { id: 1, nom: "Alice Martin", email: "alice@mail.fr", actif: true },
  { id: 2, nom: "Bob Dupont", email: "bob@mail.fr", actif: false },
  { id: 3, nom: "Charlie Durand", email: "charlie@mail.fr", actif: true },
  { id: 4, nom: "Diana Morel", email: "diana@mail.fr", actif: false },
]

À faire :

  • a) Arrow function getActifs(liste) → retourne uniquement les contacts actifs
  • b) Arrow function formatContact({ nom, email }) → retourne la chaîne "[nom] <[email]>" (utilise le destructuring dans les paramètres)
  • c) Arrow function ajouterContact(liste, nouveauContact) → retourne un nouveau tableau avec le contact ajouté à la fin (sans modifier liste)
  • d) Arrow function mettreAJour(liste, id, changementsObj) → retourne un nouveau tableau où le contact correspondant à id est mis à jour avec les propriétés de changementsObj (spread sur l'objet existant)
  • e) Utilise ces 4 fonctions dans un enchaînement : ajoute un 5e contact, mets Alice en inactif, puis affiche tous les actifs formatés

CONTRAINTES

— Toutes les fonctions sont des arrow functions avec retour implicite si possible
— Aucune mutation des données originales (immuabilité)
— Destructuring dans les paramètres de formatContact

EX 06

Modules ES6 — import / export

challenge

Les modules ES6 permettent de diviser le code en fichiers réutilisables. Pour tester, crée deux fichiers (dans un dossier local ou avec un bundler comme Vite) :

// Fichier : utils.js
// À compléter :
// 1. Export nommé : une fonction carre(n)
// 2. Export nommé : une fonction somme(a, b)
// 3. Export nommé : un tableau COULEURS = ["rouge", "vert", "bleu"]
// 4. Export par défaut : une fonction saluer(prenom)
// Fichier : main.js
// À compléter :
// 1. Importe saluer (export default) depuis utils.js
// 2. Importe { carre, somme, COULEURS } depuis utils.js
// 3. Utilise chaque import et affiche les résultats

Alternative sans bundler : teste les modules directement dans le navigateur avec <script type="module"> dans un fichier HTML.

CONTRAINTES

— Un seul export default par fichier
— Les exports nommés utilisent export const ...
— Essaie de renommer un import : import { carre as puissance2 } from './utils.js'
